像素级交叉熵：语义分割中的公式与 PyTorch

目标：看懂 [N,C,H,W] 与 [N,H,W]，并把普通交叉熵推广到整张分割掩码。

1. 一个像素就是一个分类样本

普通三分类时，一张图片只得到一个类别。语义分割时，一张 H×W 图像会得到 H×W 个类别：每个像素都要判断它是背景、道路、车辆或其他类别。

关键变化：分类对象从“一张图”变成“图中的每一个像素”；每个像素的损失仍是 -log(真实类概率)。

2. 公式只是多加了像素下标

L[n,h,w] = -log p[n, y[n,h,w], h, w] L = 1/M · Σ(n,h,w) L[n,h,w]

模型在每个像素位置输出 C 个 logits，经过 softmax 后得到 C 类概率。训练时，通常对所有未忽略像素求平均。

3. 把维度对齐

images  : [N, 3, H, W]
logits  : [N, C, H, W]  # 模型输出，未做 softmax
mask    : [N, H, W]     # 真实类别编号，dtype=torch.long

loss = torch.nn.CrossEntropyLoss()(logits, mask)
例：N=2, C=4, H=W=256 时，模型为每张图的 65,536 个像素各输出 4 个分类分数；标签只需在每个像素放一个数字 0 到 3。

4. one-hot mask 到底是什么

假设一张 2×2 图像有 3 类，原始标签图是 [[0, 2], [1, 0]]，形状为 [H,W]=[2,2]。这表示左上像素是第 0 类、右上是第 2 类。

类别编号 mask:                 one-hot mask 的 3 个通道:
[[0, 2],                       第 0 类: [[1, 0], [0, 1]]
 [1, 0]]                       第 1 类: [[0, 0], [1, 0]]
                               第 2 类: [[0, 1], [0, 0]]

one-hot 后，原先每个像素的一个整数，变成每个像素的 3 个 0/1 数字，形状从 [N,H,W] 变为 [N,C,H,W]。

默认硬标签训练：CrossEntropyLoss(logits, mask) 需要的是左边的类别编号图，因而不用手动转 one-hot。它会按每个整数标签取出对应类别的损失。

5. 最常见的两个错误

  1. 先做 softmax：CrossEntropyLoss 已经在内部完成，直接传 logits。
  2. 无必要地把 mask 做成 one-hot：普通硬标签用 [N,H,W] 的整数类别图即可；转 one-hot 会多占 C 倍左右的标签内存，也更容易出现维度或类型错误。

例外是标签本身就是“软标签”，例如 label smoothing 或类别概率图；这时才会传入形状为 [N,C,H,W] 的浮点目标。它不是普通分割标注的默认形式。

6. 二分类分割要换损失

logits = model(images)           # [N, 1, H, W]
mask = mask.float()              # [N, 1, H, W]，值为 0 或 1
loss = torch.nn.BCEWithLogitsLoss()(logits, mask)

二分类只有“前景/背景”时，使用 BCEWithLogitsLoss。不要用通道数为 1 的 CrossEntropyLoss。
