从公式到 PyTorch：交叉熵损失

目标：会写公式、会算一个样本的损失，并能避免 PyTorch 中最常见的 softmax 错误。

1. 先只看真实类别

三分类模型给一张图片的预测概率是：

p = [0.10, 0.70, 0.20]

若真实类别是第 2 类（编号为 1），我们只关心它对应的概率 p_t = 0.70。这次预测的损失是：

L = -log(p_t) = -log(0.70) ≈ 0.357
直觉：真实类别概率接近 1 时，-log(p_t) 接近 0；真实类别概率接近 0 时，损失急剧增大。因此，模型若非常自信地预测错，会受到很大惩罚。

2. 从 one-hot 形式看公式

第 2 类的 one-hot 标签为 y = [0, 1, 0]。通用多分类公式是：

L = -Σ y_i log(p_i)

代入后，标签中为 0 的项全部消失：

L = -[0·log(0.10) + 1·log(0.70) + 0·log(0.20)] = -log(0.70)

所以考试中常把多分类交叉熵直接写为 -log(p_t)。

3. 二分类是同一个思路

L = -[y log(p) + (1-y) log(1-p)]
  • 真实标签 y=1：只剩 -log(p)。
  • 真实标签 y=0：只剩 -log(1-p)。

4. 对应 PyTorch 代码

import torch
import torch.nn as nn

logits = torch.tensor([[1.2, 2.1, 0.4],
                       [0.3, 0.1, 1.7]])  # [批大小 N, 类别数 C]
targets = torch.tensor([1, 2])             # 每个样本的真实类别编号

loss = nn.CrossEntropyLoss()(logits, targets)
关键对应：代码中的 logits 还不是概率；CrossEntropyLoss 内部会执行数值稳定的 softmax 和取对数。不要先手动调用 softmax。targets 用类别编号，而不是 one-hot 向量。
