语义分割第 3 课

双线性插值：把分割 logits 放大回像素图

目标：理解一个新像素如何从四个邻点得到数值，并知道在语义分割里什么该用 bilinear、什么必须用 nearest。

1. 它在分割网络的哪一步？

输入图像                 解码器输出 logits             上采样后 logits            最终类别图
[N, 3, 512, 512]  ->  [N, C, 128, 128]  ->  [N, C, 512, 512]  ->  argmax(dim=1)

网络常在较小的空间分辨率上计算，以减少成本。每个位置的 C 个 logits 是连续分数，因此可以平滑地估计中间位置的值。双线性插值就是这一估计方式。

先记住这一条：对 logits、概率图、特征图使用双线性插值；对真实标签 mask 使用最近邻插值。

4. 为什么叫“双线性”？

先横向

在上边的 F00 和 F01 之间按 x 插一次；下边的 F10 和 F11 之间也插一次。

再纵向

在刚得到的“上方结果”和“下方结果”之间，按 y 再插一次。

等价公式

P=(1-x)(1-y)F00+x(1-y)F01+(1-x)yF10+xyF11

权重总和永远是 1。采样点离某个角越近，该角的权重越大；落在正中央时，四个角各占 0.25。

5. 分割中该对谁插值？

模型 logits

[N,C,h,w]

用 bilinear

每个数字是连续分类分数，4.2 这样的中间结果有意义。

预测类别图

argmax(logits)

通常先放大 logits，再 argmax

让所有类别通道分别被平滑放大，再决定每个像素的最大类。

真实标签 mask

[N,H,W]，值为 0、1、2...

只能用 nearest

类别编号是离散含义。0 和 2 之间的 1.3 并不代表一个真实类别。

6. PyTorch 的常用写法

import torch.nn.functional as F

# 模型输出低分辨率 logits: [N, C, 128, 128]
logits_small = model(images)

# 预测或需要与原图大小对齐时：插值 logits，再分类
logits = F.interpolate(
    logits_small,
    size=images.shape[-2:],
    mode="bilinear",
    align_corners=False,
)
pred = logits.argmax(dim=1)  # [N, H, W]

# 若必须缩放真实类别标签：只用 nearest
mask_small = F.interpolate(
    mask.unsqueeze(1).float(),
    size=logits_small.shape[-2:],
    mode="nearest",
).squeeze(1).long()
训练时要先对齐尺寸：CrossEntropyLoss 要求 logits 的空间尺寸和 mask 一致。可选择放大 logits，或把 mask 用最近邻缩小；具体取决于你的网络和损失计算位置。

关于 align_corners=False

它规定缩放时像素坐标如何对齐。PyTorch 的双线性插值通常显式写成 False；更重要的是训练、验证、推理都要保持同一设置。不要在不同阶段混用，否则边界位置可能有细小偏移。
