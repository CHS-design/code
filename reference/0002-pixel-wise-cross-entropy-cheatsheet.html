像素级交叉熵速查卡

一句话：把图像中每个像素都当成一次 C 类分类，计算损失后再对有效像素求平均。

多分类语义分割公式

L = -1/M · Σ(n,h,w) log p[n, y[n,h,w], h, w]

n 是批内图像，h,w 是像素位置，y[n,h,w] 是该像素的真实类别编号，M 是参与计算的像素总数。

PyTorch 的形状

张量形状类型与含义
logits[N, C, H, W]float；每个像素对 C 个类别的原始分数
mask[N, H, W]long；每个像素的类别编号，如 0、1、2
prediction[N, H, W]logits.argmax(dim=1) 得到的预测类别图

多分类写法

criterion = torch.nn.CrossEntropyLoss(ignore_index=255)
loss = criterion(logits, mask)  # logits: [N,C,H,W], mask: [N,H,W]

不要先对 logits 做 softmax；CrossEntropyLoss 会在类别维 C 上内部完成这一步。255 是常用的忽略标签示例，必须与你的数据集约定一致。

one-hot 说明：普通硬标签不需要转 one-hot：[N,H,W] 的整数类别图即可。one-hot 会变成 [N,C,H,W]，只在 soft label 或类别概率标签等场景使用。

二分类不是 C=1 的 CrossEntropyLoss

criterion = torch.nn.BCEWithLogitsLoss()
loss = criterion(logits, mask.float())
# logits 与 mask 的形状通常都是 [N,1,H,W]

只有一个输出通道时，softmax 恒为 1，不能区分前景与背景。二分类应使用 BCEWithLogitsLoss。
