# 项目长期记忆：研究生周报撰写要求

本项目用于维护研究生周报。后续撰写、整理、修改周报时，应始终遵守以下规则。默认只写本周实际发生、实际学习、实际完成的内容，不主动补写计划、收获或总结类模块。

## 优先级原则

撰写周报时，组内的具体要求是第一原则，学长周报模板和既有周报成品是第二原则。后续所有周报应先满足“周日提交、时间倒序、自顶向下、写清实际工作、只写本周实际内容”等硬性要求，再参考学长模板和既有成品的标题、模块、表述方式和内容组织。若具体要求与模板风格存在冲突，应以具体要求为准。
学长模板只用于参考整体风格和表达方式，不作为固定目录或固定题材清单；既有成品只用于参考简洁格式和语气，不作为扩写理由。每周内容应以当周真实学习和实际工作为中心灵活组织。

## 组内具体要求（第一原则）

1. 每周周末需要对上一周的工作进行小结，并在周日内提交周报，提交方式为发送到本微信群。

2. 周报必须采用自顶向下的写法。每个小点先用一句简洁的话概括核心内容，然后再展开说明具体工作、问题、进展或结果。

3. 周报文档按时间倒序维护。每周新增的最新周报内容应放在文档最前面。

4. 周报应清楚写明本周实际完成的工作，便于导师阅读后更有效地交流和指导。

5. 后续周报默认不写“下周计划”“下一步计划”“阶段性收获”“阶段性总结”“本周总结”“后续安排”等计划或总结类模块。除非用户明确要求，否则周报只围绕本周实际完成的学习、代码复现、项目进展、实验结果和问题分析组织内容。

6. 后续周报不要写得太细，不要使用明显的 AI 总结腔。早期学习阶段按正常学生的简洁周报来写，重点说明本周学了什么、练了什么、掌握了哪些基础内容即可，避免过多展开原理、作用、意义和后续铺垫。

## 默认周报格式

默认参考 `C:\Users\admin\Desktop\陈浩升周报.docx` 的简洁格式：

1. 每周新增内容写在文档最前面，标题使用“本周学习进展（日期范围）”。

2. 标题下面直接进入本周实际学习条目，优先使用“一. …… / 二. …… / 三. ……”这类短标题。标题应对应本周真实学习内容，例如 Python 基础、NumPy、机器学习基础、论文阅读、代码复现或实验问题。

3. 每个条目下面要分具体小点写清本周实际做了什么、学到了什么、练习了什么或观察到了什么问题，不要把多个内容合成一整段。分点默认使用 `1.    内容`、`2.    内容` 这类编号格式；在 DOCX 中编号后要保留明显空格、Tab 或悬挂缩进，让编号和正文分开。

4. 默认使用纯黑文字、普通段落和简洁标题，不使用表格、蓝色装饰、复杂封面或过度排版。主标题“本周学习进展（日期范围）”必须加粗，小标题“一. …… / 二. …… / 三. ……”必须加粗。只有用户明确要求，或本周确实有需要对比展示的实验指标时，才加入表格或图。

5. 如果本周只有学习内容，就只写学习内容；如果本周没有实验、论文或项目进展，不要虚构这些模块。

6. 周报 DOCX 修改完成后，默认只做结构校验和内容位置检查，不再进行 LibreOffice/`soffice` 可视化渲染；只有用户明确要求检查页面排版时才进行渲染。

2. 各周之间使用明显分隔线区分，并保持最新一周在最前面。

3. 在满足自顶向下要求的前提下，每周正文可使用编号列表或“一、二、三”模块组织。简单周报可直接用 1、2、3 编号；内容较多时可用“学习内容 / 代码复现 / 实验结果 / 问题分析”等一级模块，再在每个模块内用 1)、2)、3) 展开。无论采用哪种层级，每个小点都应先用一句话概括核心内容，再展开说明。

4. 内容优先写实际工作进展，包括论文阅读、方法设计、代码实现、数据集支持、模型训练、实验结果、指标变化、对比实验、失败现象和原因分析。

5. 论文阅读类内容应写清论文题目、会议或年份，并按照“背景问题、核心方法、关键机制、实验结论、与自己研究的关系”的顺序概括。若一周阅读多篇论文，可按方向归类，例如优化层、目标层、架构层，或基础模型、蒸馏方法、分割模型等。

6. 实验或方法类内容应尽量写清“本周做了什么改动、在哪些数据集或模型上验证、结果如何、本周观察到什么问题或判断”。不要在默认周报中延伸成计划段。

7. 实验结果可以配表格或关键指标。表格前要先用一句话说明实验目的或结论，表格后要解释指标变化说明了什么，避免只贴结果不分析。

8. 涉及模型结构、论文架构、思维导图、对比表时，可以插入图或表，但图表应服务于文字说明。图表前说明“为什么放这张图”，图表后说明“从图表得到什么判断”。

9. 周报语气以研究进展汇报为主，允许写当前方法效果不佳、指标未提升、模块过于复杂、跨数据集表现不稳定等真实问题，但默认只写本周已经形成的判断和影响，不主动扩展后续改进安排。

10. 早期或学习阶段的周报可侧重“学习内容 + 代码复现 + 项目进展”；进入研究推进阶段后，应更多采用“论文研读 + 方法设计 + 实验结果 + 失败原因分析”的结构。

11. 写失败实验时，建议按照“现象 -> 排除或怀疑的原因 -> 本周判断 -> 对当前工作的影响”的顺序写。例如指标不如 baseline、只在某个数据集有效、蒸馏模块过复杂但收益不足，都应明确写出本周已有判断。

12. 除非用户明确要求，周报不要单独列出“下周计划”“下一步计划”“阶段性收获”“阶段性总结”“本周总结”或“后续安排”。当本周出现实验失败、方向调整或阶段性结论时，可在具体工作条目或“问题分析”中写清当前判断和影响，但不要单独设置总结或计划模块。

13. 周报应避免空泛描述，尽量出现具体对象：论文名、模型名、数据集名、模块名、指标名、实验设置、训练轮次、对比方法、观察到的现象和自己的判断。
